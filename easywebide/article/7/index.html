Android System WebView 版本过低或不可用的解决方案
2021.2.18 修改后发布 | Treep
关注事项
  • 本篇所提及的解决方案仅适用于 Android 5.0 及以上用户。部分适用于 Android 7.0 及以上用户。
  • 我们在 EasyWeb IDE 2.5.0 版本中加入了 Android System WebView 检测机制，这意味着旧版本用户可能遇到不友好的崩溃页面，但此篇文章仍可为旧版本用户带来帮助。

    • 什么是 Android System WebView ?
    Android System WebView 是 Android OS 的一个可供其他 Android 应用调用的组件。在 Android 5.0 及更高版本，你可以通过系统应用商店更新此组件。EasyWeb IDE 非常依赖于此组件，当 Android System WebView 出现问题时，EasyWeb 的部分功能可能无法正常使用甚至崩溃。
    提示 “你的系统组件版本过低”
    当前系统的 Android System WebView 长期未更新，其版本低于 80.x。
    当前浏览器内核版本检测报告

    Android System WebView 在 Android 4.4 及以上版本为单独的系统组件，你可以点击这里或在其他位置下载 Android System WebView 高版本安装包，并使用软件包安装程序更新。
    请注意，你的系统可能无法直接更新 Android System WebView，请参照以下方案。
    对于 Android 6.0 或以下用户：你可以点此查阅解决方案。
    对于第三方厂商系统：为保证安全性，其可能限制了 Android System WebView 的升级，请尝试关闭系统优化。
    若仍无法解决问题，可在「功能配置与支持」对话框中暂时禁用 [启动应用时检测 Android System WebView 并发出警告]。
    提示 “EasyWeb 无法获取你的 Android System WebView 版本” 或 “EasyWeb 无法使用 Android System WebView”
    我们猜测你可能正在使用非原生或类原生系统；或手动更改了 “WebView 实现” 设置；或安装了 Google Chrome。

    若您为 Android 7.0 或以上用户，你可以尝试如下操作。
    打开 “系统设置”，点击 [系统信息] 或 [关于] 按钮，连续多次点击 [系统版本] 或 [OS 版本] 项直至弹出 “你已经是开发者” 提示。
    进入 “开发者选项” 页面，点击 [WebView 实现] 按钮，选择高版本的 [Android System WebView] 项。
    仍无法解决问题?
    推荐你加入 “EasyWeb IDE 用户交流” 群，更方便地交流你所遇到的麻烦。加入用户交流群。
